## 🎯 TÓM TẮT VẤN ĐỀ

**Bug:** POST `/api/game/farm/harvest` → 500 Internal Server Error
**Root cause:** Drizzle ORM generate sai SQL syntax khi insert với `.defaultNow()` trong schema

---

## 🔍 NGUYÊN NHÂN CHÍNH XÁC

### Error message (trước fix):
```
PostgresError: column "id" does not exist
Failed query: insert into "inventory" ("id", "user_id", "item_id", "quantity",
  "harvested_at", "expires_at", "acquired_at", "updated_at")
  values (default, $1, $2, $3, $4, $5, default, default)
```

### Vấn đề:
1. Schema có `.defaultNow()` trên `harvestedAt`, `acquiredAt`, `updatedAt`
2. Drizzle generate SQL với **`default`** keyword trong VALUES clause
3. PostgreSQL **KHÔNG hỗ trợ** syntax này → Insert fail → 500

### Tại sao `.defaultNow()` gây lỗi:
- Drizzle interprets `.defaultNow()` là: "insert `default` keyword, DB sẽ tự generate"
- PostgreSQL chỉ cho phép `DEFAULT` trong CREATE TABLE, không trong INSERT VALUES
- Kết quả: Syntax error → column "id" does not exist (do query fail trước khi check id)

## ✅ GIẢI PHÁP ĐÃ ÁP DỤNG

### 1. Schema modification (`src/modules/game/schema/inventory.ts`)

**Trước:**
```typescript
harvestedAt: timestamp('harvested_at', { withTimezone: true }).defaultNow().notNull(),
acquiredAt: timestamp('acquired_at', { withTimezone: true }).defaultNow().notNull(),
updatedAt: timestamp('updated_at', { withTimezone: true }).defaultNow().notNull(),
```

**Sau:**
```typescript
// MỚI — Harvest tracking (KHÔNG có defaultNow() để tránh Drizzle generate sai SQL)
harvestedAt: timestamp('harvested_at', { withTimezone: true }).notNull(),
expiresAt: timestamp('expires_at', { withTimezone: true }), // NULL = không hết hạn (shop items)

// Timestamps (KHÔNG có defaultNow() để tránh Drizzle generate sai SQL)
acquiredAt: timestamp('acquired_at', { withTimezone: true }).notNull(),
updatedAt: timestamp('updated_at', { withTimezone: true }).notNull(),
```

**Đổi:** Bỏ `.defaultNow()` để Drizzle không generate `default` keyword

---

**Trước:**

**Đổi:** Thêm `acquiredAt`, `updatedAt` vào insert values

## 🎓 BÀI HỌC & LESSONS

### What went wrong:
1. **Drizzle ORM limitation:** `.defaultNow()` generates invalid SQL syntax
2. **Testing gap:** Không test harvest flow trước khi merge code
3. **Database defaults misunderstanding:** Default values nên set ở DB level (SQL raw), không ORM level

### Best practices learned:
1. ✅ **TRÁNH `.defaultNow()`** trong Drizzle schema khi dùng `.insert()`
2. ✅ **TEST END-TO-END** flow: plant → wait → harvest → inventory → sell
3. ✅ **SET EXPLICIT VALUES** thay vì rely on ORM defaults
4. ✅ **USE SQL MIGRATIONS** cho default values (cleaner & more predictable)

### Alternative solution (better):
Thay vì set trong code, có thể dùng SQL migration để set DB-level defaults:

```sql
-- Trong migration file
ALTER TABLE inventory
  ALTER COLUMN harvested_at SET DEFAULT NOW();
  ALTER COLUMN acquired_at SET DEFAULT NOW();
  ALTER COLUMN updated_at SET DEFAULT NOW();
```

Sau đó có thể omit columns khi insert:
```typescript
await db.insert(inventory).values({
  // harvestedAt omitted → DB uses default
  // acquiredAt omitted → DB uses default
  // updatedAt omitted → DB uses default
  userId,
  itemId: plot.plantTypeId,
  quantity: 1,
  expiresAt,
});
```
